# Intro
Le présent projet concerne une application web d'administration par lots des utilisateurs, groupes et projets d'une plateforme Gitlab dédiée aux administrateurs de la plateforme.

## Ressources du projet
Les ressources du projet sont fournies dans le dépôt lui-même :

- Le cahier des charges décrivant les exigences attendues
- La configuration du serveur Ubuntu hébergeant la plateforme Gitlab de test
- Les bonnes pratiques du codage JavaScript à respecter
- Le code source des pages HTML de la version initiale

## Découpage du projet
Le projet est décomposé en 3 lots (milestones) :

1. Etude de l'expression des besoins
2. Réalisation et test des exigences fonctionnelles
3. Intégration et bilan

Les lots regroupent plusieurs tickets (issues) relevant de taches d'étude, documentation, installation, configuration, réalisation, test, etc. **Chaque ticket donnera lieu à un compte-rendu attaché au ticket et à une Merge Request s'il s'agit de code source à réaliser.**
